# Canvas Writer Agent PRD

## 背景与目标
Canvas Workspace 需要一个稳定的写入入口，避免多 Agent 直接落盘导致风格漂移与冲突。Canvas Writer Agent 专职负责读写当前画布（`canvas.json` + `notes/`），生成与更新各类文档。

目标：
- 统一文档产出（PRD、技术方案、会议纪要、总结/复盘）。
- 限定写入范围，只作用于当前 workspace。
- 保持文档结构一致、可追溯。

## 用户与使用场景
用户：当前操作者本人。

场景：
- 让 Agent 产出 PRD、技术方案、会议纪要等文档。
- 让其他 Agent 提出写作需求，由 Writer Agent 落盘。

## 范围
In scope:
- 读取当前 workspace 的 `canvas.json` 与 `notes/`。
- 新建文档并写入 `notes/`。
- 更新已有文档（追加或替换段落）。

Out of scope:
- 跨 workspace 写入。
- 复杂的多人协作锁机制。
- 自动同步到外部系统。

## 功能需求（MVP）
1) 读取当前画布
- 读取 `canvas.json` 与 `notes/` 目录内容。
- 识别已有 note 列表与基础元信息。

2) 写入文档（新建/更新）
- 支持文档类型：PRD、技术方案、会议纪要、总结/复盘。
- 新建：基于模板生成完整文档。
- 更新：对指定文档进行增量编辑。

3) 结构化模板
- 每种文档类型固定大纲结构。
- 支持简版/完整版两种输出。

4) 安全与约束
- 只读写当前 canvas 目录。
- 不修改其它路径文件。

## 非功能需求
- 可控性：写入必须明确文档名或新文档标题。
- 可追溯：记录 `updated_at` 或在正文标注更新时间。
- 简洁：1-2 句指令即可完成写入。

## 数据结构约定（简版）
`canvas.json`:
- `workspace_name`
- `notes_index`（可选）：记录标题、路径、类型、updated_at

`notes/*.md`:
- 文档头部包含标题、类型、创建/更新时间。

## 交互示例
- 指令：写一个 PRD，主题是 Canvas Writer Agent，简版。
  - 结果：创建 `notes/PRD-Canvas-Writer.md`
- 指令：更新刚才的 PRD，补充用户故事和非目标。
  - 结果：对指定章节追加或替换。

## 验收标准（MVP）
- 能读取当前画布并列出已有 notes。
- 能创建并写入至少 1 种文档类型（PRD）。
- 能对已有文档进行更新，并保留更新时间。
- 只在 canvas 目录内读写。

## 风险与应对
- 风险：多 Agent 写冲突
  - 方案：仅 Writer Agent 允许落盘。
- 风险：文档风格不一致
  - 方案：固定模板 + 简版/完整版。

## 里程碑
- M1：读取画布 + PRD 新建
- M2：支持多文档类型
- M3：增量更新与更新时间记录
