# React MF Note Plugin Demo

This folder is intentionally outside `src/plugins`. It simulates a user-owned
React plugin package that Pulse Canvas can load from Settings -> Canvas Plugins.
It uses the official `@module-federation/vite` plugin to expose the Canvas
renderer plugin module.

## Try It In Pulse Canvas

1. Build the remote after installing this demo's own dependencies:

   ```bash
   cd apps/canvas-plugin-react-mf-note-demo
   pnpm install --ignore-workspace
   pnpm build
   ```

2. Open Pulse Canvas.
3. Go to Settings -> Canvas Plugins.
4. Choose this folder:

   ```txt
   apps/canvas-plugin-react-mf-note-demo
   ```

The host reads `manifest.json`, then loads `dist/mf-manifest.json`.
This repository keeps the built `dist/` output in the demo folder, so you can
also load it immediately before changing the source. `mf-manifest.json` is
generated by the Module Federation Vite plugin and references the remote entry
and assets.

## Create A Demo Node

This demo contributes the node type `demo.note`. You can ask the Canvas Agent to
create a plugin node with:

```json
{
  "type": "plugin",
  "title": "Demo Note",
  "width": 380,
  "height": 280,
  "data": {
    "pluginId": "demo-note",
    "nodeType": "demo.note",
    "payload": {
      "title": "External React plugin",
      "body": "Loaded from a user-configured local directory.",
      "accent": "#2383e2",
      "pinned": false
    }
  }
}
```

The current host MVP loads the renderer side from external directories. Main
process read/write/action capabilities for third-party packages are a later
sandboxing step.

## Project Shape

```txt
manifest.json
package.json
vite.config.ts
src/
  NoteNodeView.tsx
  plugin.tsx
  preview.tsx
  types.ts
dist/
  mf-manifest.json
  remoteEntry.js
```

`vite.config.ts` exposes the Canvas renderer plugin through Module Federation:

```ts
federation({
  name: 'pulse_canvas_demo_note',
  filename: 'remoteEntry.js',
  manifest: true,
  exposes: {
    './plugin': './src/plugin.tsx',
  },
});
```

The Module Federation React Bridge `createBridgeComponent` is for exporting a
whole remote React application with mount/unmount lifecycle. This demo exposes a
Canvas renderer plugin object instead, so it uses the build plugin's `exposes`
API directly.
